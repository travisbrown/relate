Visualization Tool Help

Welcome to the Network Visualization Tool. Use the tree at the side to navigate through this help, or simply scroll up and down.



Mouse Modes

Picking

This mouse mode allows one to simply click to select nodes. Shift-clicking allows one to select multiple nodes, as does clicking, holding, and dragging a rectangle across the nodes one would like to select. Selected nodes show up as red, and nodes that the selected node(s) are connected to show up as yellow. One can zoom in this mode using the mouse wheel (roll up to zoom out, roll down to zoom in).

Transforming

In this mode, nodes may not be selected. Clicking and dragging translates the graph in the direction of mouse movement. One can rotate the graph by shift-clicking and dragging. One can zoom in this mode using the mouse wheel (roll up to zoom out, roll down to zoom in).



Layouts

Layout Options

Circle: positions nodes equally spaced on a regular circle dependent on node size.
Force: iterative layout that usesFruchterman-Reingold force-directed algorithm to position nodes. Nodes try to push each other apart, while edges have attractive forces.

Kamada-Kawai: Implements the Kamada-Kawai algorithm for node layout.

Spring: an iterative layout without a maximum number of iterations. Treats each edge as a spring, allowing nodes to be dragged and then spring back into place. Nodes are originally assigned random X and Y positions on the graph.

ISOM: Implements a self-organizing map layout algorithm, based on Meyer's self-organizing graph methods.



Magnitude

Number of Connections

Shows the number of unique connections a node has to other nodes within that grouping (family/non-family/all).

Centrality Measures

Degree: the number of links incident upon a node (i.e., the number of ties that a node has) divided by . Degree is often interpreted in terms of the immediate risk of node for catching whatever is flowing through the network (such as a virus, or some information).
Betweenness: a measure of what fraction of shortest paths the node/edge is present on. In other words, the higher between centrality a node/edge has, the more essential it is to traversing a graph in the shortest way possible.



Which Nodes Are Shown?

Family/Non-Family/All

Family: Filters out and shows only nodes that are in the writer's close family.
Non-Family: Filters out and shows only nodes that are not in the writer's close family.

All: Shows all nodes
